Tecnologias utilizadas

  • Boostrap 4
  • Angular 9
  • JQuery
  • SweetAlert 2
  • Localstorage

Patrones de diseno

  • Ruteo de Angular
  • Separacion de modulos de la APP principal
  • Separacion modular de servicios
  • Menu dinamico desde el backend
  • Sanitizacion de ficheros mediante pipes personalizados
  • Seguridad mediante JWT para el login y backend
  • Seguridad mediante Angular Guards para navegacion local

Pendientes

El Backend con el que trabaja fue hecho en nodejs y mongoDB con el proposito de salir rapido con el prototipo y por ende como no iba a ser un backend final, no estan todos los metodos que requiere el sistema de produccion y los que estan no tienen todos los campos y controles que se requieren.

Las cosas pendientes en esta lista se refieren unicamente al frontend
  • Conectar al backend de Java una vez hecho
  • Agregar los campos faltantes a los modelos
  • Agregar los controles de flujo de tramite
  • Ajustar la seguridad de los servicios al backend de java
  • Evaluar ventajas y desventajas de lazyload
  • Generar alta de residencias mediante bsuqueda de agenda *

* El sistema de residencias da de alta el tramite con una persona automaticamente al poner la agenda en el formulario, esta es una tarea propia del backend que llama a la agenda y genera ambas cosas (la persona que hoy llamamos equipo y el tramite que hoy llamamos principal). No se hizo este procedimiento en este prototipo debido a que la complejidad de la tarea no merece la pena en un prototipo, pero si estan las altas de personas y modificaciones de tramites por separado y los metodos en el backend para agregar tramites de prueba
